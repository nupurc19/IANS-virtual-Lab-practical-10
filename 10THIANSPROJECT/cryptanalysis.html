Transposition ciphers are vulnerable to several kinds of cipher text only attacks. some of them are mentioned as belows


2. Statistical Attack :A tranposition cipher does not change the frequency of letters in cipher text, it only reorders the letters.So, the first attack that can be applied is single letter frequency analysis. This method can be useful if the length of the cipher text is long enough. We have seen this attack before.However, transposition ciphers do not preserve the frequency of diagrams and trigrams.This means that eve cannot use these tools .In fact, if a cipher does not preserve the frequency of diagrams ad trigrams but does preserve the frequency of single letters, it is probable that the cipher is a transposition cipher.


2. Brute Force Attack : one can try all the possible keys to decrypt the message . However , the number of keys can be huge where L is the length of the ciphher text. A better approach is to guess the number of columns . Eve knows that the number of column divides l. for example if length of the cipher text is 20 characters , then 20=1*2*2*5. This means the number of columns can be combination of these factors (1,2,4,5,10,20). However, the first is out of question and the last one is unlikely.
